# LOG Instructions

## Purpose
- Create a new log file whenever starting a new project.
- File name format: `[project name] - logs.md`

## Required First Step
- Ask the user for the project name first before creating the project log file.
- At the start of any new chat, read this file first and continue using these same logging rules.

## Logging Rule
- Log every prompt from the user.
- Newest log entry must always be at the top.
- NEVER EDIT previous logs.
- If an old issue is fixed later, do not edit the old log. Add a new entry and reference it (example: `found a fix for log 0001`).

## Log Entry Format
- log number - 0001
- date and time
- exact prompt of mine (EXACT WORDS)
- summary of what you did (no need to be lengthy, only when needed)
- issues encountered (indicate if fixed or not)
- fix (if there is a fix)
- files touched

## Example Entry Template
- log number - 0001
- date and time - YYYY-MM-DD HH:MM
- exact prompt of mine (EXACT WORDS) - "..."
- summary of what you did - ...
- issues encountered - ... (fixed/not fixed)
- fix - ...
- files touched - ...
